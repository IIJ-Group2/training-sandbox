# CLAUDE.md — サンドボックス喫茶

このリポジトリは Git と LLM の練習用サンドボックス。プレーンな HTML / CSS だけの静的サイトで、ビルド・フレームワーク・npm は一切使わない。作業者は Git 初心者なので、以下を厳守すること。

## 変更のルール
- 1回の作業で変更してよいのは、原則 Issue に書かれた**対象ファイル1つだけ**
- 依頼された箇所以外は書き換えない。ページ全体の作り直し・リフォーマット・「ついでの改善」は禁止
- `site/index.html` と `site/css/style.css` はどの Issue でも編集禁止（CI でも検知されて赤になる）
- 新しい技術を持ち込まない: package.json / node_modules / React / TypeScript / JavaScript / ビルドツール / 外部 CDN の追加は禁止。プレーンな HTML / CSS のみ
- 新規ファイルは Issue に指定されたファイル名・拡張子を**正確に**使う（例: `faq.html`。`faq.html.txt` は不可）

## 作業の進め方
- 作業者は最初に Issue の URL を貼ってくる想定。貼られたらその Issue を読み、対象ファイルと「できたかどうかの確認」を把握する。どの Issue か分からない場合は「どの Issue ですか？Issue のページの URL を貼ってください」と平易に聞く
- 編集の前に対象ファイルを読み、既存の構造（ヘッダー・「トップに戻る」リンク・class 名・表の形）を把握して、それに合わせる
- 編集後、変更したファイルを読み直し、タグの閉じ忘れ・相対パスのリンク切れ（`site/pages/` からは `../index.html`、画像は `../images/…`）がないか確認し、ページがちゃんと表示される状態かを確かめる
- 依頼が対象ファイル以外の変更を必要とする場合は、勝手に進めず、その旨を平易な日本語で作業者に伝えて確認を取る

## PR の作り方（重要: PR テンプレートは自動では効かない）
- PR タイトルは「Fix #<Issue 番号>: <変えたことの要約>」。Issue 番号が分からなければ作業者に聞く
- PR 本文は `.github/pull_request_template.md` を読み、**その見出し構成をすべて含めて自分で書く**（Claude Code からの PR 作成ではテンプレートが自動挿入されないため）
- 「使ったプロンプト」欄には、このセッションで**作業者から受け取ったメッセージを原文のままコピー**して載せる（要約・言い換え・省略をしない。複数回受け取った場合は全部、受け取った順に）
- 「自己確認チェック」のチェックボックスは**チェックを入れずに**（`- [ ]` のまま）載せる。確認するのは作業者本人の仕事のため

## 説明のしかた
- 作業者への説明は、専門用語を避けた平易な日本語で。「何を」「どのファイルで」「どう変えたか」を2〜3行で伝える
- 最後に必ず「PR に付く『プレビューはこちら』の URL を開いて、Issue の『できたかどうかの確認』を自分の目でチェックしてください」と案内する
