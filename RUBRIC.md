# LLM の使い方 4か条（RUBRIC）

このリポジトリでの練習テーマは「Claude に丸投げする」ではなく「**発注して、検品する**」です。レビューでもこの4つを見ます。

## 1. 1回のお願いは1つだけ。ほしい形も必ず書く
- 悪い例: 「メニュー直して、あとデザインも良くして、ついでに全体をきれいにして」
- 良い例: 「site/pages/menu.html の表に、飲み物を3つ、行（<tr>）として追加してください」
- 「表で」「HTML で」「箇条書きで」のように、**ほしい出力の形**を毎回書きます。形を書かないと、LLM は勝手に形を決めます

## 2. 出てきたものを読んで、頼んだ通りか確認してから PR にする
- 「行を3つ足して」と頼んだのにページ全体が作り直されていたら、それは事故です。「追加した3行以外は元に戻してください」と言いましょう
- モック（見た目のサンプル）を頼んだのにアプリ一式が出てきたら、そこで止める
- **確認するのはあなたの仕事です。** Claude の仕事ではありません

## 3. ファイル名と拡張子をまともにつける（つけさせる）
- HTML なら `.html`、CSS なら `.css`。`faq.html.txt` や `無題.txt` は NG
- 新しいファイルを作るときは、Issue に書いてあるファイル名を**そのまま**使う
- 頼んでいないファイル（package.json など）が勝手に増えていたら「消してください」と頼む。自動チェック（持ち込みチェック）も赤くなって、日本語で教えてくれます

## 4. 使ったプロンプトを PR に残す
- PR 本文の「使ったプロンプト」欄には、あなたが Claude に送った文が**自動で書き写されます**
- PR を作ったら、**自分の送った文がそのまま載っているか読んで確認**してください。載っていなければ、PR のコメント欄に自分でコピペすれば OK です
- 恥ずかしがらなくて大丈夫。うまくいったプロンプトはチームの財産、うまくいかなかったプロンプトは最高の教材です
