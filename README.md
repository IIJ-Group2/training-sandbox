# サンドボックス喫茶 — Git と LLM の練習場

ここは、架空の喫茶店「サンドボックス喫茶」の紹介サイトを直しながら、GitHub と Claude Code の使い方を練習するリポジトリです。

- **何を壊しても大丈夫です。** 本番のプロジェクトとは完全に切り離されています
- **プログラムの実行環境はいりません。** 結果はぜんぶブラウザで確認できます
- 公開中のサイト: https://iij-group2.github.io/training-sandbox/

## まずはこれだけ（クイックスタート）

1. **Issues** タブで Issue を1つ選び、自分を Assign する（はじめてなら「レベル1」のラベルが付いたもの）
2. [Claude Code](https://claude.ai/code) でこのリポジトリを開き、**最初に Issue の URL を貼ってから**、お願いを1つだけ書く
3. できあがった PR の「プレビューはこちら」リンクを開き、Issue の「できたかどうかの確認」を自分の目でチェックする

迷ったら、下の詳しい手順を読んでください。

## 詳しい手順

### 1. Issue を1つ選ぶ
1. このリポジトリの **Issues** タブを開きます
2. Issue には「レベル1」〜「レベル3」のラベルが付いています。**はじめての人はレベル1**からどうぞ
3. やる Issue を開いて、右側の **Assignees** で自分を選びます（「私がやります」の宣言です）
4. すでに誰かが Assign されている Issue は選ばないでください（1人1 Issue）

### 2. Claude Code (web) で直してもらう
1. https://claude.ai/code を開き、このリポジトリ（training-sandbox）を選びます
2. 先に `RUBRIC.md`（LLM の使い方 4 か条）を 1 回読んでおきます
3. チャットの最初に **Issue のページの URL を貼り**、続けてお願いしたいことを**1つだけ**書きます。Issue 本文の「プロンプトのヒント」をコピペして書き換えるのがおすすめです

### 3. Claude の返事をちゃんと読む
- 頼んだ通りのことをしていますか？
- 変えたファイルは、Issue に書いてある対象ファイル**だけ**ですか？
- 違っていたら「◯◯だけをやり直してください。他は元に戻してください」と伝えれば直してくれます

### 4. PR（プルリクエスト）を作る
1. Claude Code の **Create PR** ボタンを押します
2. PR の本文は Claude が書いてくれます。「使ったプロンプト」の欄に、**自分が送った文がそのまま載っているか**を読んで確認してください（載っていなければ、PR のコメント欄に自分でコピペすれば OK です）

### 5. 自分の目で確認する（いちばん大事）
1. 2〜3分待つと、PR に「**プレビューはこちら**」というコメントが付きます
2. リンクを開いて、Issue の「できたかどうかの確認」を**全部**自分の目でチェックします
3. PR 下部の **Checks** が緑（✓）になっているかも見ます。赤（✕）のときは ✕ の横の Details をクリックすると、**日本語で理由と直し方**が書いてあります
4. 確認できたら、PR 本文の「自己確認チェック」のチェックボックスをクリックして ✓ にします（PR の画面でそのままクリックできます）

#### 「Files changed」タブの見かた
- PR のページの上のほうに「Conversation / Commits / Files changed」というタブが並んでいます
- **Files changed** を開くと、変わったファイルの一覧と中身が見えます。緑の行が「増えた行」、赤の行が「消えた行」です
- ここで「変わっているのが Issue の対象ファイルだけか」を確認します

### 6. レビューを待つ
- レビュー担当には**自動で通知が届く**ので、こちらから何か操作する必要はありません
- 直してほしいコメントが付いたら、Claude Code の**同じセッション**で修正をお願いすれば、PR は自動で更新されます
- レビューが通ってマージされると、あなたの変更が公開サイトに反映されます 🎉

## 困ったときは
- **プレビューのコメントが付かない** → 2〜3分待ちます。それでも来なければレビュー担当に聞いてください
- **Checks が赤い** → ✕ の横の Details をクリックしてメッセージを読みます。だいたい「余計なファイルが入っている」か「変更が大きすぎる」です。メッセージに「Claude への頼み方」も書いてあります
- **Claude のセッションを閉じてしまった** → 新しい会話を開いて、**PR の URL を貼って**「この PR の続きです。◯◯を直してください」と頼めば続きができます
- **わけが分からなくなった** → その PR は Close して、新しくやり直して OK です。ここは練習場で、失敗するために存在しています

## 名前について（1つだけ注意）
このリポジトリは**公開（public）**です。Issue や PR に書いた内容は誰でも見られます。**本名ではなく GitHub のユーザー名**で活動し、PR に載せるプロンプトにも本名や個人情報を書かないようにしましょう。
