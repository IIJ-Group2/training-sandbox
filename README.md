# サンドボックス喫茶 — Git と LLM の練習場

ここは、架空の喫茶店「サンドボックス喫茶」の紹介サイトを直しながら、GitHub と Claude Code の使い方を練習するリポジトリです。

- **何を壊しても大丈夫です。** 本番のプロジェクトとは完全に切り離されています
- **プログラムの実行環境はいりません。** 結果はぜんぶブラウザで確認できます
- 公開中のサイト: https://iij-group2.github.io/training-sandbox/

## まずはこれだけ（クイックスタート）

1. **Issues** タブで Issue を1つ選び、自分を Assign する（はじめてなら「レベル1」のラベルが付いたもの）
2. [Claude Code](https://claude.ai/code) でこのリポジトリを開き、**最初に Issue の URL を貼ってから**、お願いを1つだけ書く
   （Claude Code がまだ使えない人は、下の「べつのルート: Gemini で直す」を見てください）
3. できあがった PR の「プレビューはこちら」リンクを開き、Issue の「できたかどうかの確認」を自分の目でチェックする

迷ったら、下の詳しい手順を読んでください。

## 詳しい手順

### 1. Issue を1つ選ぶ
1. このリポジトリの **Issues** タブを開きます
2. Issue には「レベル1」〜「レベル3」のラベルが付いています。**はじめての人はレベル1**からどうぞ
3. やる Issue を開いて、右側の **Assignees** で自分を選びます（「私がやります」の宣言です）
4. すでに誰かが Assign されている Issue は選ばないでください（1人1 Issue）

### 2. Claude Code (web) で直してもらう

（Claude Code がまだ使えない人は、下の「べつのルート: Gemini で直す」を見てください）

1. https://claude.ai/code を開き、このリポジトリ（training-sandbox）を選びます
2. 先に `RUBRIC.md`（LLM の使い方 4 か条）を 1 回読んでおきます
3. チャットの最初に **Issue のページの URL を貼り**、続けてお願いしたいことを**1つだけ**書きます。Issue 本文の「プロンプトのヒント」をコピペして書き換えるのがおすすめです

### 3. Claude の返事をちゃんと読む
- 頼んだ通りのことをしていますか？
- 変えたファイルは、Issue に書いてある対象ファイル**だけ**ですか？
- 違っていたら「◯◯だけをやり直してください。他は元に戻してください」と伝えれば直してくれます

### 4. PR（プルリクエスト）を作る
1. Claude Code の **Create PR** ボタンを押します
2. PR の本文は Claude が書いてくれます。「使ったプロンプト」の欄に、**自分が送った文がそのまま載っているか**を読んで確認してください（載っていなければ、PR のコメント欄に自分でコピペすれば OK です）

### 5. 自分の目で確認する（いちばん大事）
1. 2〜3分待つと、PR に「**プレビューはこちら**」というコメントが付きます
2. リンクを開いて、Issue の「できたかどうかの確認」を**全部**自分の目でチェックします
3. PR 下部の **Checks** が緑（✓）になっているかも見ます。赤（✕）のときは ✕ の横の Details をクリックすると、**日本語で理由と直し方**が書いてあります
4. 確認できたら、PR 本文の「自己確認チェック」のチェックボックスをクリックして ✓ にします（PR の画面でそのままクリックできます）

#### 「Files changed」タブの見かた
- PR のページの上のほうに「Conversation / Commits / Files changed」というタブが並んでいます
- **Files changed** を開くと、変わったファイルの一覧と中身が見えます。緑の行が「増えた行」、赤の行が「消えた行」です
- ここで「変わっているのが Issue の対象ファイルだけか」を確認します

### 6. レビューを待つ
- レビュー担当には**自動で通知が届く**ので、こちらから何か操作する必要はありません
- 直してほしいコメントが付いたら、Claude Code の**同じセッション**で修正をお願いすれば、PR は自動で更新されます
- レビューが通ってマージされると、あなたの変更が公開サイトに反映されます 🎉

## べつのルート: Gemini で直す

Claude Code の準備がまだ終わっていない人も大丈夫です。チームは Google Workspace なので、**Google アカウントさえあれば** [gemini.google.com](https://gemini.google.com) にログインするだけで今すぐ参加できます。

流れは主経路（上の「詳しい手順」）とほとんど同じで、違うのは「誰がファイルを貼ったり PR を書いたりするか」だけです。

- Issue を選ぶところ（手順1）と、自分の目で確認してレビューを待つところ（手順5・6）は**まったく同じ**です
- 変わるのは手順2〜4の部分。Claude Code の「Create PR」ボタンのような自動化が無いので、**その1〜その5**として自分の手で1つずつやります

ひとことで言うと: 今のファイルをコピー → Gemini に直してもらう → GitHub に貼って PR を作る → PR の本文を自分で書く → あとは主経路の手順5・6と同じ、という流れです。

### その1. いまのファイルの中身をコピーする
1. リポジトリの **Code** タブ →「**site**」→「**pages**」の順にクリックして進み、対象ファイル（例: `menu.html`）を開きます
2. ファイルの中身が表示された画面の右上あたりに並んでいるアイコンの中から、「**Copy raw content**」（隣に「Raw」というボタンもあります。それが目印です）をクリックします。これで中身がまるごとコピーされます
   - 手でドラッグして選択しないでください（行が欠けたり、行番号が混ざったりする事故のもとです）
   - **新しいファイルを作る Issue**（例: `faq.html` を新規作成する Issue）の場合は、コピーするのは「これから作るファイル」ではなく、Issue に書いてある「**参考にするファイル**」（例: `news.html`）です。同じ手順でそちらをコピーします

### その2. Gemini に「全文」で直してもらう
1. [gemini.google.com](https://gemini.google.com) を開き、Google アカウントでログインします
2. 下のコピペ用の型を使います。「変更:」の行には、Issue の「プロンプトのヒント」のうち**お願いの内容だけ**を書き写してください。**1行目の「この Issue をお願いします: （URL）」という行は貼らないでください**（Gemini は URL を開けないので、貼ると混乱の原因になります）。最後に、その1でコピーした中身を貼ります

   ```
   このあとに貼る HTML ファイルに、次の変更を 1 つだけ加えてください。

   変更: （ここに、Issueの「お願いする内容」の部分だけを書く。URLの行は貼らない）

   守ってほしいこと:
   - 変更後のファイルの「全文」を、ふつうのコードブロックで 1 つだけ返してください
   - Canvas やプレビュー画面は使わないでください。ふつうのコードブロックで返してください
   - 指定した変更以外は、空白や改行もふくめて 1 文字も変えないでください
   - 新しいページやファイルを勝手に作らないでください
   - 「（中略）」のような省略はせず、最初から最後まで全部書いてください

   ---ここからファイルの中身---
   （その1でコピーした中身をここに貼る）
   ```

   例（Issue #1「メニュー表に飲み物を3つ追加しよう」の場合の「変更:」の書き方）:
   ```
   変更: site/pages/menu.html のメニュー表に、飲み物を3行追加してください。内容は「カフェラテ 500円」「ゆず茶 450円」「ココア 480円」。既存の行（<tr>と<td>）の書き方に合わせてください。
   ```

3. なぜ「全文」で返してもらうか: 一部分だけ返されると「どこに貼るか」で必ず事故ります。全文をまるごと置き換えるのがいちばん安全です
4. もし画面の右側に、喫茶店のページっぽい絵（プレビュー）がドーンと開いたら、それは **Canvas** という別モードです。その状態だとコピーボタンが無く詰まります。「コード」表示に切り替えるか、「Canvas は使わず、ふつうのコードブロックで返し直してください」と頼み直してください
5. 返事を読みます（主経路の「3. Claude の返事をちゃんと読む」と同じです）: 頼んだ1箇所だけ変わっているか確認してください。Gemini が「素敵な喫茶店ページを作りますね」と言って別ページを作り始めたり、デザインまで勝手に変えていたら →「いいえ。渡したファイルのその1箇所だけ変えて、省略せずに全文を返し直してください」と伝えます
6. OK なら、Gemini の返事のコードブロック右上にある**コピーボタン**でコピーします（手で選択すると ``` や「html」という文字まで混ざるので使わないでください）

**新しいファイルを作る Issue**（`faq.html` など、Issue #5 タイプ）の場合は、代わりにこちらの型を使います:
```
site/pages/（新しいファイル名） という新しいページを1つだけ作ってください。
ページの作り（ヘッダーや「トップに戻る」リンクなど）は、次に貼る site/pages/（参考にするファイル名） と同じにしてください。

載せる内容: （Issueに書いてある内容をここに書く）

守ってほしいこと:
- ファイルは1つだけ作ってください。他のファイルは作らない・変えないでください
- Canvas は使わず、ふつうのコードブロックで返してください
- 省略せず、最初から最後まで全部書いてください

---ここから参考ファイル（site/pages/（参考ファイル名））の中身---
（その1でコピーした参考ファイルの中身をここに貼る）
```

例（Issue #5「よくある質問ページを作ろう」の場合）:
```
site/pages/faq.html という新しいページを1つだけ作ってください。
ページの作り（ヘッダーや「トップに戻る」リンクなど）は、次に貼る site/pages/news.html と同じにしてください。「トップに戻る」のリンク先は ../index.html にしてください。

載せる内容:
Q: 予約はできますか？ / A: できません。実在しないお店なので。
Q: Wi-Fiはありますか？ / A: あります。パスワードは店員にお声がけください。
Q: 勉強していってもいいですか？ / A: 大歓迎です。Gitの練習もどうぞ。

守ってほしいこと:
- ファイルは1つだけ作ってください。他のファイルは作らない・変えないでください
- Canvas は使わず、ふつうのコードブロックで返してください

---ここから参考ファイル（site/pages/news.html）の中身---
（その1で news.html をコピーした中身をここに貼る）
```

### その3. GitHub に貼り付ける

**既存のファイルを直す場合**
1. GitHub で対象ファイルを開き、右上の「**えんぴつアイコン（Edit this file）**」をクリックします
2. エディタの中をクリック → **Ctrl+A**（Mac は **⌘+A**）で全選択 → Delete → Gemini からコピーした全文を貼り付けます
3. 貼り付けたら、**一番下までスクロール**して、最後の行が元のファイルと同じ終わり方（`</html>` など）になっているか見てください。「（中略）」のような行があったら、その2に戻って「省略せずに全文を返し直して」と Gemini に頼んでからやり直します
4. 右上の緑のボタン「**Commit changes...**」をクリックします
5. 出てきた小さい画面で「**Create a new branch for this commit and start a pull request**」を選びます
   - ⚠ 「**Commit directly to the main branch**」は選ばないでください。ここがこのルートでいちばん注意するところです。もし間違えて main に直接コミットしてしまったら、自分で直そうとせず、レビュー担当に伝えてください
6. 「**Propose changes**」をクリックすると「Open a pull request」という画面に変わります。**ここではまだ緑のボタンを押しません。** 次の「その4」で PR の本文を書いてから押します

**新しいファイルを作る場合（`faq.html` など）**
1. リポジトリのトップページで「**Add file**」→「**Create new file**」をクリックします
2. ファイル名欄に、Issue に書いてあるファイル名を**そのまま**入力します（例: `site/pages/faq.html`。「/」を打つとフォルダになります）。最後が `.html` で終わっているか指さし確認してください
3. 本文欄に Gemini からコピーした全文を貼り付けます（その3-3 と同じく、最後の行が正しく終わっているか確認します）
4. 以降は既存ファイルの4〜6と同じです（Commit changes... → new branch → Propose changes。ここでもまだ緑のボタンは押しません）

### その4. PR の本文を自分で書いて、PR を完成させる
- その3の続きで「Open a pull request」という画面が出ています。タイトルは自動で入っているものでそのまま OK です
- 本文欄には、テンプレートが最初から入っています。Claude Code と違って自動では埋まらないので、自分で埋めます
  - 「使ったプロンプト」欄にある「※Claude が自動で書き写します…」という1行は消して、その2で Gemini に送った文をそのまま貼ってください。ファイル全文を貼った部分は「＋ファイル全文」と書けば省略して OK です
  - 消してしまった／どこに何を書けばいいか迷ったら、下のコピペ用テンプレを丸ごと貼って埋めてください
- 埋め終わったら、画面いちばん下の緑のボタン「**Create pull request**」を押します（隣の▼で Draft は選びません）。これで PR が完成します

コピペ用 PR 本文テンプレ:
~~~
## どの Issue ですか
Fixes #（ここに Issue の番号）

## 何を変えましたか（1〜3行、ふつうの言葉で OK）
（例: menu.html のメニュー表に飲み物を 3 行追加した）

## 使ったプロンプト（Gemini に送った文）
※Gemini ルートなので自分で貼っています

```
（Gemini に送ったお願いの文をここに貼る。ファイル全文を貼った部分は「＋ファイル全文」と書けば OK）
```

## 自己確認チェック（PR の画面でチェックボックスをクリックして、全部 ✓ にしてからレビューを待つ）
- [ ] 「プレビューはこちら」の URL を開いて、Issue の「できたかどうかの確認」を全部満たしていた
- [ ] Checks（自動チェック）が緑になっている
- [ ] 「Files changed」タブを見て、変わっているのが Issue の対象ファイルだけだった
- [ ] 「使ったプロンプト」欄に、自分が Gemini に送った文をそのまま貼った
~~~

### その5. 確認とレビューは、いつもと同じ
- 上の「5. 自分の目で確認する」「6. レビューを待つ」を、そのまま実行してください
- **このルートだけの追加チェック**: 「Files changed」タブを開いて、ファイルが**ほぼ全部赤と緑**になっていたら、Gemini が空白や改行まで変えてしまった印です。その2に戻ってやり直してください
- Checks が赤くて、Details のメッセージに「Claude に『〜してください』と頼んでください」と書いてあったら、それは Gemini ルートでは実行できません。代わりに、このセクションのその2〜その3の要領で**自分で**直してください
- **直しを頼まれたとき**: PR の「Files changed」タブ → 直したいファイルの右上にある「**...**」→「**Edit file**」をクリックすると、その PR のブランチ上のファイルが開きます。いま入っている全文をコピーし、その2と同じ型を使って（＝全文を渡し直して）Gemini の同じ会話に修正をお願いします。返ってきた全文で、その3と同じように貼り替えます。**ここでは「Create a new branch」ではなく「Commit directly to the ◯◯ branch」を選んでください**（自分の PR のブランチに直接コミットして OK です。ここで「Create a new branch」を選ぶと別の PR ができて迷子になるので注意してください）。PR は自動で更新されます
- Gemini に送る文にも、本名や個人情報は書かないでください（下の「名前について」と同じ注意です）

## 困ったときは
- **プレビューのコメントが付かない** → 2〜3分待ちます。それでも来なければレビュー担当に聞いてください
- **Checks が赤い** → ✕ の横の Details をクリックしてメッセージを読みます。だいたい「余計なファイルが入っている」か「変更が大きすぎる」です。メッセージに「Claude への頼み方」も書いてあります
- **Claude のセッションを閉じてしまった** → 新しい会話を開いて、**PR の URL を貼って**「この PR の続きです。◯◯を直してください」と頼めば続きができます
- **わけが分からなくなった** → その PR は Close して、新しくやり直して OK です。ここは練習場で、失敗するために存在しています
- **Claude Code にログインできない／準備が終わっていない** → 上の「べつのルート: Gemini で直す」で参加できます
- **間違えて main に直接コミットしてしまった** → 自分で直そうとせず、レビュー担当に伝えてください

## 名前について（1つだけ注意）
このリポジトリは**公開（public）**です。Issue や PR に書いた内容は誰でも見られます。**本名ではなく GitHub のユーザー名**で活動し、PR に載せるプロンプトにも本名や個人情報を書かないようにしましょう。
